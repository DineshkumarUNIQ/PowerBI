What is Power BI?
Understanding the big picture before diving in

Power BI is Microsoft's business intelligence and data visualization tool. It lets you take raw data from spreadsheets, databases, cloud services — and turn it into beautiful, interactive reports and dashboards — without needing to code.

🍳

Simple Analogy: Think of it like a Kitchen

Raw data = raw ingredients (vegetables, meat, spices)
Power Query = the prep work (chopping, washing)
Data Model = your recipe (how ingredients relate)
Reports/Dashboards = the finished dish you serve to others

📊

Without Power BI

You get scattered Excel files, manual copy-paste, outdated numbers, no visuals, hours of work for a single report.

✨

With Power BI

Connect once, auto-refresh data, drag-and-drop charts, share with your whole team, everything updates live.

ℹ️
Real World Use Cases Sales performance tracking · Financial reporting · Supply chain monitoring · Marketing analytics · HR dashboards · Executive KPI boards

Install Power BI Desktop — Step by Step
Get set up in under 5 minutes
  1. Go to Microsoft Store — Open the Windows Start menu → search "Microsoft Store" → search for Power BI Desktop → Click Install (it's free, ~500MB)
  2. Alternative: Download directly — Visit powerbi.microsoft.com → click "Download free" → run the .msi installer
  3. Launch Power BI Desktop — After install, open it. You'll see a splash screen with "Get started" options
  4. Sign in (optional but recommended) — Click the person icon top-right → Sign in with your Microsoft / work account. This enables publishing to the cloud later.
  5. Close the splash screen — Click the X on the "Get started" dialog. You're now looking at the blank Power BI canvas — ready to go!
⚠️
Windows Only! Power BI Desktop runs natively on Windows only. Mac users can use Parallels, a Windows VM, or use the cloud-based Power BI Service instead.
💡
System Requirements Windows 10/11 · 2GB RAM minimum (8GB recommended) · .NET 4.6 or later · Internet connection for signing in

Importing Data — Excel, CSV & SQL
How to connect Power BI to your data sources
  1. Click Home tab → click Get Data button (top-left of ribbon)
  2. A dialog appears with 100+ connectors. Choose your source: Excel Workbook, Text/CSV, SQL Server, etc.
  3. Browse to your file OR enter the SQL server name & database
  4. A Navigator window opens showing all sheets/tables inside your file. Check the boxes next to what you want to load.
  5. Click Load to import directly, OR Transform Data to open Power Query and clean first (recommended!)
Excel / CSV Tips
  • Make sure row 1 is your header row
  • Avoid merged cells in Excel — they cause problems
  • CSV = comma-separated, TSV = tab-separated
  • Power BI auto-detects data types on import
SQL Server Tips
  • You need server name + database name
  • Choose Import mode for small-medium datasets
  • Choose DirectQuery for huge live databases
  • Write a custom SQL query for filtered import
ℹ️
Import vs DirectQuery — Import copies data into Power BI (fast but needs manual refresh). DirectQuery always reads live from the database (always current but slower). For learning, always use Import mode.
🔧
Power Query Editor — Your Data Lab
This is where raw data gets cleaned and shaped
🧪

Analogy: Power Query is Like a Lab

Think of Power Query as a lab where you receive a messy specimen (raw data) and run experiments on it (transformations). Every experiment is recorded in the Applied Steps panel. You can undo any step or see exactly what happened. The original data is NEVER changed — it's non-destructive.

Fig 2.2 — Power Query Editor layout with queries panel, data grid, and applied steps
🚫

Remove Nulls

Select the column → click the dropdown arrow → Uncheck "(null)" → OK. Or: Home → Remove Rows → Remove Blank Rows.

🔄

Change Data Types

Click the data type icon (abc, 123, 📅) to the left of the column name. Choose: Text, Whole Number, Decimal, Date, Date/Time, True/False.

✂️

Split Column

Select column → Transform tab → Split Column → By Delimiter (e.g. split "John Smith" by space into "John" | "Smith").

💡
Pro Tip: Every step is an M formula! Power Query records your clicks as M language code. Click "Advanced Editor" (View tab) to see the full script. This is how experts automate complex transformations.

Relationships — Connecting Your Tables
How different tables talk to each other in Power BI
🏫

Analogy: Think of a School

Students table = one student record per student (1 row per person)
Grades table = many grade records per student (multiple subjects)
→ One student → Many grades: this is a One-to-Many relationship

Fig 3.1 — Relationship between Products (dimension) and Sales (fact) tables
1️⃣→🔢

One-to-Many (1:*) — Most Common

One record in Table A matches many records in Table B. Example: 1 Product → Many Sales. Always connect PK (Products) → FK (Sales.ProductID).

🔢→1️⃣

How to Create a Relationship

Go to Model View → drag the ProductID field from Products table → drop it onto ProductID in Sales table. A line appears. Done!

⭐
Star Schema — The Golden Standard
The best way to organize tables in Power BI

The Star Schema puts one central Fact table (with numbers/events) in the middle, surrounded by Dimension tables (with descriptive context). It looks like a star!

Fig 3.2 — Star Schema with Sales fact table at center, 5 dimension tables radiating outward
⭐
Why Star Schema? Power BI's DAX engine is optimized for star schemas. They give you fast query performance, easy-to-understand relationships, and flexible filtering. Always aim for this structure.

Introduction to DAX — Data Analysis Expressions
Write formulas to calculate business metrics
📐

Analogy: DAX is Like Excel Formulas — But Smarter

If you know Excel's =SUM(), =AVERAGE(), =IF() — you already understand 60% of DAX. The key difference: DAX is context-aware. It knows what filters are active in your visual and adjusts the calculation automatically.

Measures (Dynamic Calculations)
DAX Measure Examples
-- Total Revenue
Total Revenue = SUM(Sales[Revenue])

-- Order Count
Order Count = COUNT(Sales[SaleID])

-- Average Order Value
Avg Order = AVERAGE(Sales[Revenue])

-- Revenue for Electronics only
Electronics Rev = 
  CALCULATE(
    SUM(Sales[Revenue]),
    Products[Category] = "Electronics"
  )
Key DAX Functions Explained
  • SUM(column) — Add all values in a column
  • COUNT(column) — Count non-blank rows
  • AVERAGE(column) — Mean of the values
  • CALCULATE(expr, filter) — Evaluate an expression with additional filters applied — the most powerful DAX function!
  • SUMX(table, expression) — Calculate row-by-row and then sum results
  • IF(condition, true, false) — Conditional logic

🎯 Measure vs Calculated Column — Know the Difference!

FeatureMeasureCalculated Column
When calculatedAt query time (dynamic)At data refresh (static)
Responds to filters✔ Yes✘ No
Stored in model✘ No (computed live)✔ Yes
Use forKPIs, totals, %Row-level categories
Example=SUM(Sales[Revenue])= Sales[Price]*Sales[Qty]

Filters & Slicers — Let Users Explore the Data
Interactive controls that make reports come alive

🔍 Filter Panel

The filter pane (right side of the canvas) lets you filter at 3 levels:

  • Visual-level filters — only affect the selected chart
  • Page-level filters — affect everything on current page
  • Report-level filters — affect all pages in the whole report
ℹ️
Filters are hidden from users by default. Slicers are visible on the canvas for user interaction.

🎚️ Slicers

Slicers are visual filter controls you place on the canvas. Users click them to filter the report.

  • Dropdown slicer — compact, good for many options
  • List slicer — shows all options, multi-select
  • Date range slicer — slider for date filtering
  • Relative date slicer — "Last 7 days", "This month"
  1. Click the Slicer icon in Visualizations panel
  2. Drag a field (e.g. Year) into the "Field" well
  3. Resize and position on the canvas

🔬 Drill-Down & Drill-Through

⬇️

Drill-Down

Navigate hierarchy levels within a chart. Example: Year → Quarter → Month → Day. Click the double-arrow icon on a visual to enable drill mode, then click a bar to go deeper.

➡️

Drill-Through

Right-click a data point → jump to a detail page filtered to that selection. Example: click on "Electronics" in a chart → navigate to a dedicated Electronics detail page.

Layout Rule: Z-Pattern

Users read top-left → top-right → bottom-left → bottom-right. Put your most important KPIs top-left, summary charts top-right.

Use Themes

Get your report into the cloud and in front of your team
Fig 5.2 — From Desktop to Service to Team: the sharing workflow

Publishing Your Report — Step by Step

  1. In Power BI Desktop → click Home tab → click Publish button (top-right area)
  2. A dialog asks: "Select a destination" → choose your Workspace (default is "My workspace" for personal use)
  3. Click Select. Power BI uploads your .pbix file to the cloud. Takes 10–60 seconds.
  4. A success message appears with a link: "Open [filename] in Power BI" → click it to see your report live in the browser
  5. To share: In the Service → open your report → click Share → enter colleague email addresses → click Send
  6. To export PDF/PPT: In the Service → File menu → Export to PDF / Export to PowerPoint
🔄
Schedule Auto-Refresh In Power BI Service → Datasets → Settings → Scheduled Refresh. Set it to refresh daily at 6 AM so your data is always up to date when people arrive at work.
🔒
Pro License Required for Sharing To share with others outside "My workspace", both you AND the recipient need Power BI Pro ($10/user/month) or a Premium capacity workspace.
